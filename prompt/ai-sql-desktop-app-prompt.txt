BUILD A CROSS-PLATFORM AI-ASSISTED SQL DATABASE DESKTOP APP

You are an expert product+tech team (software architect, UX lead, senior frontend, senior backend, DevOps, QA) tasked with delivering a production-grade, commercial desktop application from scratch. Produce code, configuration, tests, build pipelines, packaging, and documentation.

0) OVERVIEW

Create a cross-platform desktop app (macOS, Windows, Linux) for managing SQL databases. Core goal: users can ask in natural language for actions or questions; an AI engine turns those requests into SQL that appears in the central editor and can be executed safely with strong guardrails.
	•	Initial DB engine: Microsoft SQL Server (must be first-class).
	•	Architecture must be open to future engines (e.g., PostgreSQL).
	•	The UI layout mirrors Azure Data Studio / VS Code:
	•	Left: collapsible Tree (servers → databases → schemas → tables/views/procs → columns, indexes, keys, constraints).
	•	Center: Work Area with flexible tiling (split panes, tabs, drag-to-dock bottom/right/side-by-side). Top sub-toolbar with Run/Explain/Format/Cancel, DB/connection pickers, and row limit.
	•	Bottom of the Center: resizable Results Panel (grid, messages, execution time, row count, warnings). User can drag splitter to show more editor or more results.
	•	Right: resizable Chat Panel (AI assistant like Copilot/Cursor) that generates SQL or actions; requests materialize as editable SQL in the current editor tab.
	•	Top: native menu (File, Edit, View, Database, AI, Help) with standard shortcuts.
	•	Dialogs for adding/editing DB connections, preferences, telemetry, updates.
	•	Local persistent store for settings, connections, history, chat logs.

1) PLATFORM & STACK (choose defaults and implement)
	•	Shell: Electron (latest stable) with React + TypeScript UI, Vite bundler.
	•	State: React Query + Zustand (or Redux Toolkit) with clear layering.
	•	Styling: Tailwind CSS + headless UI components. Provide both Light/Dark themes.
	•	Icons: Lucide; monospaced editor font with ligatures; respect OS high-DPI and accessibility.
	•	Code Editor: Monaco Editor configured for SQL (T-SQL, later others). Include formatting, lint hints, bracket matching, intellisense hooks.
	•	Backend (app process): Node.js (TypeScript) for portability and easy MCP embedding.
	•	DB Drivers:
	•	SQL Server: mssql (tedious). Provide pooled connections, parameterized queries, cancel tokens.
	•	Abstraction layer that can support future engines (define IDatabaseProvider).
	•	Local DB: SQLite via better-sqlite3 for settings, connections, chat history, query history, saved snippets. Use a small ORM like Drizzle (SQL schema first). Encrypt sensitive fields (see Security).
	•	AI Providers (configurable): pluggable provider interface supporting Azure OpenAI, OpenAI, Anthropic, local Ollama. Users supply API key + endpoint per provider. Store provider profiles in local DB (encrypted secrets).
	•	MCP (Model Context Protocol): prepare to embed MCP tools in the Node backend (stdio transport). Include examples of at least:
	•	db.execute (safe execution wrapper)
	•	db.describe (schema/introspection)
	•	file.read/write (workspace snippets)
	•	Design clear JSON schemas for tool inputs/outputs.

2) SECURITY, PRIVACY & GUARDRAILS (mandatory)

Implement a full guardrail system around AI-generated SQL:
	•	Read-only mode default for AI actions. Separate “Allow Writes” toggle per connection and per session.
	•	Before executing any DDL/DML, show a Review & Confirm modal with:
	•	Parsed statement list, targeted objects, estimated row counts (dry-run), transaction preview.
	•	Option to wrap in BEGIN TRAN … ROLLBACK/COMMIT with configurable timeout.
	•	Schema allowlist per connection (databases/schemas/tables the AI can touch).
	•	Blocklist for dangerous operations: DROP DATABASE, TRUNCATE, cross-db writes, ALTER LOGIN, etc. Require manual override.
	•	Static SQL lint (SQLFluff ruleset) and parameterization by default. Never interpolate user strings unsafely.
	•	Execution limits: max rows, max time, cancel button wired to driver.
	•	Audit log (local DB): who/when/connection, SQL text (hash), result summary.
	•	Secret storage: encrypt credentials in local store using OS keychain where possible (Keytar: Keychain on macOS, DPAPI on Windows, libsecret on Linux). Never log secrets.

9) KEY IMPLEMENTATION DETAILS
	•	IDatabaseProvider interface:
	•	connect(config), disconnect(), listDatabases(), listSchemas(db), listTables(db, schema), getTableInfo(db, schema, table), execute(sql, params, options), explain(sql), estimateImpact(sql).
	•	First implementation: SqlServerProvider using mssql with pooled connections and cancel.
	•	AI Provider interface: generate(messages, tools, model, apiKey, endpoint). Support function calling / tool use. Configurable models per provider.
	•	MCP embedding: start MCP server inside Node process; expose tools: describe_schema, search_schema, propose_sql, explain_sql, format_sql, simulate_write, execute_readonly. Transport: stdio. Provide JSON Schema for each tool.
	•	Schema Context: maintain per-connection snapshot (names only; optional samples). Build a small local vector index for table/column descriptions to ground prompts.
	•	CSV Export: stream to file, UTF-8 with BOM option, configurable delimiter and date format.
	•	Accessibility: keyboard nav for tree/editor/results/chat; screen reader labels; high contrast theme.
